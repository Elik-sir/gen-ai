# Отчёт по ДЗ (Семинар 4)

## 1. Корпус

Корпус собран из транскрипта лекции, нарезан на 10-минутные документы (`lecture_10min_*.txt`) и использован для RAG-оценки.

- Фактический размер корпуса: 17 документов
- Общий объём: 72,007 символов
- Общий объём: 11,160 слов

## 3. Результаты (hit-rate@5)

Конфигурации:

- **A (fixed-size):** `chunk_size = 2000`, без overlap
- **B (recursive):** `chunk_size = 400`, `overlap = 80`

fixed-size - 0.67 (10.0 / 15)
recursive - 0.73 (11.0 / 15)

## 4. Анализ ошибок (3 кейса)

### Кейс 1: вопрос `id=2` (перефраз)

- **Fixed:** MISS (0.00)
- **Recursive:** OK (1.00)
- **Что происходит:** ответ про «почему агенты стали массовыми» состоит из нескольких фрагментов внутри окна `lecture_10min_002`; при fixed в топ не попадал нужный кусок, при recursive короткие семантические фрагменты лучше матчились на перефраз.

### Кейс 2: вопрос `id=7` (multi-hop)

- **Fixed:** MISS (0.00)
- **Recursive:** OK (1.00)
- **Что происходит:** вопрос требует собрать несколько условий («когда агент не нужен» + «что выбрать вместо»). Для таких композитных формулировок recursive чаще вытаскивает релевантные локальные куски, чем крупные fixed-чанки.

## 5. Вывод

На данном корпусе выиграла стратегия **recursive**: `0.73` против `0.67` у fixed-size.  
Преимущество особенно заметно на перефразах и multi-hop вопросах, но на отдельных вопросах с объединённым контекстом из двух источников fixed может быть стабильнее.
