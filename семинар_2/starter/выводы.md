# Выводы

Сгенерировано 50 валидных заявок. Для борьбы с mode collapse использовано квотирование по 10 городам и сиды на связку специальность-курс. По городам верхняя доля составила 10.0% (топ: Краснодар), по специальностям — 14.0% (топ: аналитик).

Квотирование по городам сделало распределение заметно стабильнее: разброс по городам (max-min) = 3 заявок, по специальностям (max-min) = 1 заявок. Связка speciality -> desired_course снизила число случайных комбинаций и уменьшала перекос в сторону определённых курсов

Про валидатор: в итоговом датасете нарушений правила `city in CITIES` нет. Это означает, что либо модель сразу соблюдала ограничение, либо невалидные ответы отсеивались и исправлялись на этапе `response_model + max_retries`. Явных инцидентов, когда финальный объект нарушал это правило, не осталось.
